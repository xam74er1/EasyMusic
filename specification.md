# Improv Playlist Application - Technical Specification

## Overview
A web application designed to help improvisation theater groups create, manage, and download music/video playlists. The application features a chatbot powered by Gemini that acts as an intelligent assistant to search for videos, suggest tracks, and automatically update the playlist using function calling.

## Core Features

### 1. Playlist Management
- **Video Entities**: Each video in the playlist tracks:
  - Name of the music / Title
  - Author / Creator
  - YouTube Link
  - Category (e.g., drama, pop music, happy music)
  - Speed (e.g., slow, medium, fast)
  - Tags (custom identifiers)
  - Duration
- **Manual Editing**: All fields for any video can be easily edited manually through the UI.
- **Ordering & Filtering**: The playlist can be sorted and filtered by category, speed, and standard tags.

### 2. AI Chatbot Assistant
- **Gemini Integration**: Connects to the Gemini API using an API key provided in the backend.
- **Function/Tool Calling**: The chatbot is equipped with tools to actively perform tasks such as searching for videos, editing the playlist, adding new entries, and categorizing tracks based on the conversation.
- **Natural Language Editing**: Users can simply tell the chat, "Add a fast, happy pop song," and the bot will search and populate the list.

### 3. YouTube Search & Integration
- **YouTube API**: Uses a YouTube API key to search for specific tracks directly.
- **Direct Linking**: Maintains direct links to the source videos for quick access.

### 4. 4 and  Downloader
- **Batch Downloading**: A dedicated feature to download all videos in the playlist that have not yet been downloaded.
- **Backend Execution**: To ensure security and performance, no downloading occurs on the frontend. The backend handles the file retrieval and acts as the bridge.

### 5. Local MP3 Import & Analysis
- **Drag & Drop Interface**: Users can drop single or multiple MP3 files, as well as ZIP archives generated by the app (to share data between instances), into the application.
- **Metadata Analysis**: Automatically analyze dropped MP3s to determine the song name and artist.
- **AI-Assisted Tagging**: Optional AI assistance (deactivated by default) to enhance metadata. If the AI fails, the process continues gracefully.
- **Batch Validation**: When dropping multiple MP3s or a ZIP archive, a popup window appears to allow the user to validate the names and all parameters before comprehensively saving them to the database.

### 6. Application Navigation & Routing
- **Tab Organization**: Re-organize the excessive tabs into a cleaner, more manageable layout.
- **Routing System**: Implement a router (e.g., React Router) so every tab and view has a distinct, navigable URL route.

### 7. Library Manager Enhancements
- **Track Download**: Provide an option to download individual MP3s with their proper names directly from the library manager.
- **Folder Archive**: Allow downloading a full folder as a ZIP archive for easy offline access to tracks.

### 8. Profile Management
- **Profile-Linked Logic**: Rework the profile system to ensure that all distinct application features (Playlist, Virtual DJ, Simplified Virtual DJ, etc.) are linked to the active user profile, isolating data and settings.

## Technical Stack Architecture

### Frontend
- **Framework**: React built with Vite.
- **Styling**: Pure, plain CSS. No CSS frameworks (like Tailwind or Bootstrap). The styling should rely on clean, modern, component-scoped CSS.
- **Architecture**: Strict component-based design (e.g., `ChatBox`, `PlaylistTable`, `VideoRow`, `EditModal`) to maintain highly readable and clean code.

### Backend
- **Language**: Python (FastAPI or Flask recommended for API handling).
- **Security & External APIs**: ALL external API calls (Gemini API, YouTube Data API) are strictly executed on the backend to protect API keys.
- **Core Responsibilities**:
  - Provide endpoints for frontend communication (`/api/chat`, `/api/videos`, `/api/download`).
  - Manage interactions with Gemini's tool-calling format.
  - Use processing libraries (like `yt-dlp` or `pytube`) to handle the heavy lifting of video downloads.
  - Store playlist state (using a lightweight database like SQLite or simply a JSON file for the MVP).
